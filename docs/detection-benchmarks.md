## 4. Synthetic Generator Parameters

### Human session model
- **Reaction time**: `lognormal(μ=5.7, σ=0.3×modifier)` — median ~300ms, range 150–800ms
- **Stick movement**: Brownian walk toward random target with jitter proportional to speed
- **Trigger**: ramp + micro-tremor (3-8 Hz, ~2% amplitude) — models physiological tremor
- **IMU**: Brownian motion + movement-correlated acceleration + thermal drift
- **Fatigue**: timing variance increases over session duration

### Macro session model
- **Timing**: constant 50ms interval (CV = 0.0)
- **IMU**: zero (controller not physically moved)
- **Detection signal**: L5 CV < 0.08 + zero IMU

### Aimbot session model
- **Right stick**: instantaneous snap to random target (zero transition frames at aggression=1.0)
- **Left stick**: normal human movement
- **IMU**: normal (controller is physically held)
- **Detection signal**: L3 right-stick jerk distribution

### Injection session model
- **Sticks**: smooth sinusoidal (software-generated, plausible)
- **IMU**: zero (dead giveaway — injected HID doesn't include real sensor data)
- **Detection signal**: L4 Mahalanobis (IMU features near-zero while sticks active)

### Warmup attack model
- **Progress**: humanity_proxy linearly increases from 0.1 → 0.8 across N sessions
- **IMU**: starts near-zero, gains noise as attack progresses (mimics warming up)
- **Detection signal**: BehavioralArchaeologist correlated positive slopes

## 5. Path to Real-World Benchmarks

To replace synthetic benchmarks with real-world data:

1. **Connect DualShock Edge via USB**: `scripts/hardware_setup.sh`
2. **Capture baseline sessions** (minimum 10): `python scripts/capture_session.py --duration 60`
3. **Run threshold calibrator**: `python scripts/threshold_calibrator.py sessions/*.json`
4. **Apply calibration profile** to bridge: set `calibration_profile.json` path in config
5. **Re-run benchmark tests** with real data — update this document with empirical figures
6. **Test at each skill tier** (bronze through diamond) with N≥20 sessions per tier
7. **Test each attack type** with N≥10 attack sessions

Minimum acceptable targets for production deployment:
- False positive rate < 5% at all human skill tiers
- Detection rate > 95% for macro/injection attacks (clear signals)
- Detection rate > 80% for aimbot/warmup attacks (ambiguous signals)
- All thresholds must have 95% confidence intervals from N≥50 real sessions

### 7.2 Threshold Calibration — Magic Numbers vs. Hardware Reality

| Threshold | Current Magic Number | Hardware-Derived (N=4) | Delta | Implication |
|-----------|---------------------|----------------------|-------|-------------|
| L4 Mahalanobis anomaly | 3.0 | **18.064** | **+502%** | Magic number would produce false positives on every real session |
| L4 Mahalanobis continuity | 2.0 | **12.043** | **+502%** | Same as above |
| L5 CV threshold | 0.08 | **0.124** (P10 human) | +55% | Human P10 is above current bot threshold — may need adjustment |
| L5 entropy threshold | 1.5 bits | **0.122** (P10)* | -92% | *Computed from inter-report intervals (wrong denominator — see note) |
| Stick noise floor (LSB std) | 5.0 | **31.96** | **+539%** | Real sticks at rest are 6.4× noisier than synthetic assumption |
| IMU gyro noise floor (LSB std) | 50.0 | **407.91** (P95) | **+716%** | Real IMU at 1000 Hz is 8.2× noisier than synthetic assumption |

> **Note on L5 entropy calibration**: The threshold_calibrator.py computes L5 metrics from
> inter-report intervals (1ms at 1000 Hz). The L5 oracle uses inter-*press* intervals (100–1000ms).
> These are different quantities. The L5 threshold must be calibrated from sessions with
> ≥20 intentional button-press events per session.

### 7.3 IMU Noise Profile (Real Hardware)

| Axis | Mean (LSB) | Std (LSB) | vs. Injection Threshold (20 LSB) |
|------|-----------|----------|--------------------------------|
| gyro_x | -1.1 | **292.3** | 14.6× above threshold |

> These measurements were taken with the controller resting on a desk (minimal movement).

### 7.4 Critical Pre-Production Actions Required

1. **Recalibrate L4 Mahalanobis thresholds** with N≥50 active gaming sessions.
   Current magic numbers (3.0/2.0) will produce near-100% false positives on real sessions.
2. **Validate L5 CV/entropy thresholds** with sessions containing N≥20 intentional presses.
   Use `capture_session.py` during competitive gameplay (not resting).
3. **Update stick noise model** from 5.0 → ~32 LSB in BiometricFusionClassifier.
   Current covariance matrix assumption (5 LSB) is off by 6.4×.
4. **Validate injection threshold** (currently 20 LSB gyro std): hardware confirms
   14.6× margin on gyro_x during resting — appears conservative and sound.
